# ZYNQ Core v7.5.29 Documentation

ZYNQ Core is a modern, lightweight, and powerful game library loader with intelligent device detection. It lets you dynamically load, search, filter, sort, randomize, and manage large collections of games with a clean and chainable API.

---

## Introduction

ZYNQ Core v7.5.29 is designed to make large game libraries feel like a living, breathing catalog you can sculpt in real time. It supports filtering, chaining, device awareness, and persistent user preferences.

### What's New in v7.5.29

* Added `shuffle()` method to randomize the current list
* Added `sortBy(key)` and `sortByPopularity()` methods
* Added `getRandomOne()` shortcut
* Added `getByAlias()` and `getByCategory()` lookup methods
* New history system: `addToHistory`, `getHistory`, `clearHistory`
* New getters: `total` and `filteredCount`
* Improved chaining behavior and stability

## Quick Start

### 1. Initialize the library

```javascript
await ZYNQ.init();
```

### 2. Get the full game list

```javascript
console.log(ZYNQ.list);
```

### 3. Get one random game

```javascript
const game = ZYNQ.getRandomOne();
console.log(game.name);
```

### 4. Chained random selection

```javascript
const game = ZYNQ.random().list[0];
console.log(game.name);
```

## Configuration

You can configure ZYNQ Core during initialization using `init()`.

| Option   | Type   | Default  | Description                                 |
| -------- | ------ | -------- | ------------------------------------------- |
| `mode`   | string | `'all'`  | `'all'`, `'supported'`, or `'favs'`         |
| `sort`   | string | `'name'` | Sort key: `'name'`, `'category'`, `'alias'` |
| `search` | string | `''`     | Initial search query                        |
| `src`    | string | default  | Custom JSON database URL                    |
| `cdn`    | string | default  | Custom thumbnail CDN base URL               |

### Example

```javascript
await ZYNQ.init({
  mode: 'supported',
  sort: 'category',
  search: 'mario',
  src: 'https://your-site.com/games.json'
});
```

## Core API

### Properties

| Property             | Type   | Description                                            |
| -------------------- | ------ | ------------------------------------------------------ |
| `ZYNQ.list`          | Array  | Current filtered, sorted, or randomized game list      |
| `ZYNQ.total`         | Number | Total number of games in the library                   |
| `ZYNQ.filteredCount` | Number | Number of games after current filters                  |
| `ZYNQ.deviceType`    | Number | 1=Desktop(touch), 2=Desktop(mouse), 3=Mobile, 4=Tablet |

### Methods

#### Filtering & Search

* `search(query)` → Filter games by name or alias
* `filterSupported()` → Keep only supported games
* `filterFavorites()` → Keep only favorite games
* `reset()` → Reset all filters

#### Sorting & Randomization

* `sortBy(key)` → Sort by name, category, alias, etc.
* `sortByPopularity()` → Sort by play count
* `shuffle()` → Randomize current list
* `random(limit)` → Get random games
* `getRandomOne()` → Get a single random game

#### Lookup

* `getByAlias(alias)` → Get game by alias
* `getByCategory(category)` → Filter by category

#### Favorites

* `toggleFavorite(alias)` → Toggle favorite state
* `isFavorite(alias)` → Check favorite state

#### History (v7.5.29)

* `addToHistory(alias)` → Add game to history (max 50)
* `getHistory()` → Retrieve history list
* `clearHistory()` → Clear history

### Chaining Example

```javascript
const games = ZYNQ
  .search('sonic')
  .filterSupported()
  .shuffle()
  .sortBy('name')
  .list;
```

## Data Structure

Each game object in `ZYNQ.list` follows this structure:

```javascript
{
  name: "Super Mario Bros",
  category: "Platformer",
  alias: "mario",
  url: "platformer/mario",
  thumb: "https://cdn.../platformer/mario.webp",
  devices: [2, 3, 4],
  isSupported: true,
  isFavorite: false
}
```

* `isSupported` is dynamically evaluated based on device compatibility
* `isFavorite` is stored in LocalStorage
* All thumbnails are standardized at 400x300px

## Usage Examples

### Random Supported Game

```javascript
const game = ZYNQ
  .filterSupported()
  .random()
  .list[0];

console.log(game.name);
```

### Get 5 Random Games from Category

```javascript
const racingGames = ZYNQ
  .getByCategory('racing')
  .random(5)
  .list;
```

### Favorites Workflow

```javascript
ZYNQ.toggleFavorite('mario');

const favorites = ZYNQ
  .reset()
  .filterFavorites()
  .list;
```

### History Tracking

```javascript
ZYNQ.addToHistory(game.alias);

const recent = ZYNQ.getHistory();
```

## Advanced Features

### Device Detection

ZYNQ automatically detects device type to optimize game compatibility.

| Value | Type                        |
| ----- | --------------------------- |
| 1     | Desktop with touch          |
| 2     | Desktop with mouse/keyboard |
| 3     | Mobile phone                |
| 4     | Tablet / large touch        |

Example:

```javascript
if (ZYNQ.deviceType === 3 || ZYNQ.deviceType === 4) {
  console.log('Touch device detected');
}
```

---

### Favorites System

Favorites persist automatically via LocalStorage.

```javascript
ZYNQ.toggleFavorite('mario');

console.log(ZYNQ.isFavorite('mario'));

const favorites = ZYNQ
  .reset()
  .filterFavorites()
  .list;
```

---

### History System

Track recently viewed or played games.

```javascript
ZYNQ.addToHistory('mario');

const history = ZYNQ.getHistory();

ZYNQ.clearHistory();
```
